# 데일리 리서치 로그 양식 (기사 + 논문 통합)

기사든 논문이든 동일한 양식으로 기록합니다. **자료유형** 필드로만 구분합니다.
항목이 **1건 이상**이면 맨 위에 **"오늘의 요약 (Daily Executive Summary)"**을 추가합니다.

**파일명 권장:** `daily_YYYY-MM-DD.md` (Cursor·GitHub가 날짜순으로 읽기 쉽도록)

> 자동 파이프라인(`python -m src.main daily-catchup`)이 생성하는 md와 **동일 양식**을 따릅니다.
> 수동 보충 시에도 아래 구조를 맞추면 월간 보고서 파싱(`daily_markdown_loader.py`)에 그대로 사용할 수 있습니다.

---

## 파일 상단 메타데이터 (파일 1개 = 하루 1개)

```
# 데일리 리서치 로그

날짜: YYYY-MM-DD
기록자:
총 항목 수: N건 (기사 n / 논문 n)
신뢰도 분포: A n건 / B n건 / C n건
```

---

## 오늘의 요약 (Daily Executive Summary)

> **1건 이상**일 때 작성. 자동 생성 시 키워드 관련도(직접/간접) 표 + GitHub 앵커 링크 형식.

```
## 오늘의 요약 (Daily Executive Summary)

**분석 기준 키워드 (상위 3개):** 키워드1 · 키워드2 · 키워드3

**오늘의 공통 흐름:** (하루 전체를 관통하는 테마 한 줄)

오늘 수집 N건 (출처1, 출처2 …)

**항목별 핵심 요약 (키워드 연결 포함):**

| 관련도 | 항목 | 한 줄 요약 |
|--------|------|-----------|
| **직접** | [기사 제목…](#anchor-slug) | 구체적 사실 → **[키워드]** 직접 연관 |
| **간접** | [기사 제목…](#anchor-slug) | 육하원칙(누가·어디·무엇·언제·수치) 기반 구체적 사실 → **[키워드]** 간접 연관 (이유) |

**눈여겨볼 신호 (키워드 관점):**
- **[키워드]** …

- **상충되는 정보:** (해당 없음) 또는 충돌 설명

---
```

**수동 작성 시 주의:**
- 표의 `항목` 열은 `[제목](#slug) · [원문](url)` 형식 — `#slug`는 본문 점프, `원문`은 외부 기사
- 표의 `항목` 열 링크 `[제목](#slug)`와 본문 `<a id="slug"></a>` + `### N. 제목`이 **같은 slug**를 써야 GitHub에서 점프 가능
- 항목의 **출처**·**링크/DOI**는 `[라벨](url)` 마크다운 링크로 적어 원문을 바로 열 수 있게 함
- “중요성 강조”, “빠르게 발전”, “의문을 불러일으켰음” 등 **사실·수치 없는 추상 문장**은 Executive Summary에 넣지 않음
- 한 줄 요약은 **누가·어디·무엇·언제·수치** 중 3가지 이상 포함 (시장조사 스캔용)

---

## 항목 기록 (항목당 1블록, 하루에 여러 개 반복)

```
<a id="anchor-slug"></a>
### 1. 제목

- **자료유형:** 기사 / 논문 / 보고서(시장조사) / 공식발표(IR·정책) / 기타
- **출처:** [매체명·저널명·기관명](https://원문-url)
- **저자/발행기관:** (논문일 경우 저자명, 기사일 경우 매체)
- **발행일:** YYYY-MM-DD
- **링크/DOI:** [원문](https://원문-url)
- **요약:**
  - **개요:** … (-함/-임 명사형 종결)
  - (해석) …
- **신뢰도:** A / B / C (기준 하단 참조)
- **태그:** #태그1 #태그2 (최대 4개)
- **비고:** (선택) 분석 키워드 · 키워드 관련도(직접/간접) · 매칭 키워드
```

## 태그 분류체계 (월간 보고서 챕터와 매칭)

| 태그 | 의미 |
|------|------|
| #기술 | 신기술, R&D, 기술 발표 |
| #논문 | 학술 연구 결과 (성능, 실험, 방법론) |
| #투자 | 투자 라운드, 밸류에이션, 펀딩 |
| #M&A | 인수합병, 전략적 제휴 |
| #제품출시 | 신제품, 서비스 런칭 |
| #기업동향 | 조직, 인력, 실적, 전략 |
| #규제 | 법안, 정책, 표준 |
| #경쟁 | 경쟁사 비교, 점유율, 포지셔닝 |
| #시장수치 | 시장규모/성장률 추정치 |
| #리스크 | 사고, 논란, 부정적 전망 |
| #전문가전망 | 애널리스트/전문가 예측 |

여러 태그 중복 가능 (예: `#논문` `#기술`)

## 신뢰도 등급 기준

- **A (높음):** 정부·공공기관 원문(korea.kr·부처 .go.kr 보도자료), 연합뉴스·뉴시스 1차 보도, 공공 R&D·정책기관(KISTEP·KIET·KIPO·KETEP·한국전력 등), 국내 학술지·국책연구원 동료심사 논문
- **B (중간):** 국내 경제·IT 전문매체(헤럴드경제·동아일보·전자신문·ZDNet Korea 등), 2차 인용·전문자료 요약, 기업·공기업 IR/보도자료
- **C (참고):** 익명 소스, 추측성 기사, 단순 재가공 콘텐츠, 미검증 블로그 — **자동 생성 시에는 C가 나오지 않음. 수동 기록 시에만 사용**

<a id="acme-ai-series-c"></a>
### 1. Acme AI, 시리즈 C 2억 달러 유치

- **자료유형:** 기사
- **출처:** TechCrunch
- **저자/발행기관:** TechCrunch 편집부
- **발행일:** 2026-06-15
- **링크/DOI:** https://example.com/article
- **요약:**
  - Acme AI가 시리즈 C로 2억 달러 유치, 기업가치 12억 달러로 평가됨
  - 주도 투자사는 XYZ Capital, 기존 투자자 ABC Ventures도 참여
  - (해석) 동일 분야 투자가 3개월 내 3건째로, 섹터 과열 신호일 가능성
- **신뢰도:** B
- **태그:** #투자 #경쟁
- **비고:** 분석 키워드: AI infrastructure · 키워드 관련도: 직접
